# Team Context

팀의 공유 지식을 AI 코딩 에이전트에 마운트하는 시스템. 이 템플릿을 fork하고, 팀 지식을 채우면 에이전트가 자동으로 참조합니다.

## 왜 필요한가

AI 코딩 에이전트는 매 대화를 백지에서 시작합니다. 팀 컨벤션, 용어, 프로젝트 구조, 담당자를 모릅니다. **Team Context**를 마운트하면 에이전트가 팀 지식을 즉시 이해합니다.

## 작동 방식

```
┌─────────────────────────────────┐
│  팀 레포 (GitHub/GHE)            │
│  conventions/ glossary/ ...     │
└──────────────┬──────────────────┘
               │  mount
               ▼
┌─────────────────────────────────┐
│  ~/.claude/team-context/{id}/   │
│  로컬 캐시 + CLAUDE.md 주입      │
└──────────────┬──────────────────┘
               │  자동 참조
               ▼
┌─────────────────────────────────┐
│  AI 에이전트가 팀 지식을           │
│  대화 중 자동으로 활용             │
└─────────────────────────────────┘
```

1. 이 템플릿 레포를 **fork**
2. 팀 문서를 **채우기** (컨벤션, 용어집, 프로젝트, 서비스)
3. Claude Code에서 **마운트**:
   ```
   이 URL을 보고 팀 컨텍스트를 설치해줘: https://[your-pages-url]/SETUP.md
   ```
4. 에이전트가 팀 지식을 자동으로 참조

## 문서 구조

| 디렉토리 | 용도 |
|-----------|---------|
| `glossary/` | 팀 고유 용어, 약어, 내부 시스템명 |
| `conventions/` | 코딩 컨벤션, 네이밍 규칙, 브랜치 전략 |
| `projects/` | 현재 프로젝트, 에픽, 스프린트 |
| `services/` | 서비스 구조, 의존성, 오너십 |
| `docs/` | 온보딩 가이드, 프로세스 문서 |

## 동작

| 명령 | 설명 |
|---------|--------------|
| **mount** | SETUP.md URL로 팀 컨텍스트 설치 |
| **unmount** | 마운트된 팀 컨텍스트 제거 |
| **update** | 원격 레포에서 최신 내용 가져오기 |
| **push** | 내용 추가/수정 후 공유 레포에 반영 |

## 외부 소스 동기화

`.context/sources.yml`을 추가하면 Confluence, Jira, GitHub에서 자동 동기화:

```yaml
sources:
  - id: team-wiki
    type: confluence
    space: YOUR_SPACE_KEY
    sync_to: docs/
```

전체 템플릿: [sources.example.yml](.context/sources.example.yml)

## 가드레일

시크릿, 개인정보, 개인 의견, 조직 민감 정보 수집을 방지하는 안전 규칙이 내장되어 있습니다. **사람이 아니라 시스템/프로세스를 개선하는 방향으로 기록합니다.**
