# Team Context Guardrails

push 전 반드시 확인한다.

## 절대 수집 금지

- **시크릿**: API 키, 토큰, 비밀번호, `.env` 값, 인증서
- **PII**: 실명+연락처, 주민번호, 계좌번호, 고객 실제 데이터
- **개인적 내용**: 개인 평가, "누구 때문에~", 갈등/불만/핀잔/비난, 1on1, 특정인 귀책
- **조직 민감**: 퇴사/이동 예정, 비공개 인사 결정, 팀 간 정치
- **저작권**: 유료 서비스 전문 복붙, 무단 타팀/타사 문서

## 수집 전 확인

| 항목 | 확인 |
|------|------|
| 팀 합의 여부 | 개인 의견인가, 팀 합의인가? |
| 다른 팀 정보 | 우리 관점 인터페이스인가, 남의 내부 사정인가? |
| 실제 데이터 | 마스킹/익명화 되었는가? |
| 유효 기간 | 시한부면 `expires` 표기했는가? |

## 올바른 표현

사람이 아닌 시스템/프로세스를 개선하는 방향으로 기록한다.

| 금지 | 허용 |
|------|------|
| "철수가 빼먹어서 장애남" | "로깅 누락 → 체크리스트 추가" |
| "영희는 리뷰가 늦음" | "리뷰 SLA: 24시간 이내" |
