대화 중 팀 전체에 유용한 지식이 나오면 추가를 제안한다.
직접 push하지 않고, 제안만 한다.

## 제안 기준

다음에 해당하면 제안한다:
- 팀이 합의한 컨벤션이나 규칙
- 새로 정의된 용어나 약어
- 아키텍처/설계 결정과 그 이유
- 반복될 수 있는 트러블슈팅 해결법
- 배포/테스트/리뷰 프로세스 변경

다음은 제안하지 않는다:
- 개인 취향이나 의견
- 일회성 디버깅 내용
- 이미 팀 컨텍스트에 있는 내용

## 제안 형식

```
💡 이 내용을 팀 컨텍스트에 추가하면 좋겠습니다:
  • 내용: {한 줄 요약}
  • 위치: {추천 경로, 예: conventions/api-naming.md}

추가하려면 "팀 컨텍스트에 추가해줘"라고 말씀하세요.
```

## 규칙

- 한 대화에서 같은 내용을 두 번 제안하지 않는다.
- 제안은 간결하게. 대화 흐름을 끊지 않는다.
- 사용자가 무시하면 더 이상 권하지 않는다.
